Framework-Specific Questions (Part 4)

// Section 1: React

61. What is the virtual DOM and how does it work in React?
    Answer: Virtual DOM is a lightweight copy of the real DOM.
    React uses it to optimize updates by comparing it with the real DOM.

62. Explain the component lifecycle in React
    Example:
    - Mounting: constructor, render, componentDidMount
    - Updating: shouldComponentUpdate, render, componentDidUpdate
    - Unmounting: componentWillUnmount

63. How do you manage state in a React application?
    - useState for local state
    - useReducer for complex state logic
    - Context API for global state
    - Redux for large-scale applications

64. What are hooks in React?
    Example:
    - useState: const [count, setCount] = useState(0);
    - useEffect: useEffect(() => { /* side effects */ }, [dependencies]);
    - useContext: const value = useContext(MyContext);

// Section 2: Vue.js

66. What are Vue.js directives and how are they used?
    Example:
    - v-if: conditionally render elements
    - v-for: render a list of items
    - v-bind: bind attributes
    - v-model: two-way data binding

67. Explain Vue.js lifecycle hooks
    Example:
    - beforeCreate, created
    - beforeMount, mounted
    - beforeUpdate, updated
    - beforeDestroy, destroyed

68. How does data binding work in Vue.js?
    Answer: Vue.js uses reactive data binding with a virtual DOM.
    Changes in data automatically update the DOM.

69. What is Vuex and how is it used?
    Example:
    import Vuex from 'vuex';
    const store = new Vuex.Store({
        state: { count: 0 },
        mutations: { increment(state) { state.count++; } }
    });

70. How do you handle routing in Vue.js?
    Example:
    import VueRouter from 'vue-router';
    const router = new VueRouter({
        routes: [
            { path: '/home', component: Home },
            { path: '/about', component: About }
        ]
    });

// Section 3: Angular

71. What is the difference between a component and a directive in Angular?
    Answer: Components are directives with a template.
    Directives are used to manipulate the DOM.

72. Explain Angular's dependency injection
    Example:
    @Injectable({
        providedIn: 'root'
    })
    export class MyService {
        constructor(private http: HttpClient) {}
    }

73. How does Angular handle forms?
    - Template-driven forms
    - Reactive forms

74. What is RxJS and how is it used in Angular?
    Example:
    import { of } from 'rxjs';
    of(1, 2, 3).subscribe(console.log);

75. Explain Angular routing
    Example:
    import { RouterModule, Routes } from '@angular/router';
    const routes: Routes = [
        { path: 'home', component: HomeComponent },
        { path: 'about', component: AboutComponent }
    ];
    @NgModule({
        imports: [RouterModule.forRoot(routes)],
        exports: [RouterModule]
    })

// Section 4: State Management

76. What is Redux and how does it work?
    Example:
    import { createStore } from 'redux';
    const reducer = (state = {}, action) => {
        switch (action.type) {
            case 'INCREMENT':
                return { ...state, count: state.count + 1 };
            default:
                return state;
        }
    };
    const store = createStore(reducer);

78. How do you use MobX for state management?
    Example:
    import { observable, action } from 'mobx';
    class Store {
        @observable count = 0;
        @action increment() { this.count++; }
    }

79. What are the benefits of using state management libraries?
    - Centralized state
    - Predictable state changes
    - Easier debugging
    - Improved scalability

80. How do you handle state in a large-scale application?
    - Use Redux or MobX
    - Split state into modules
    - Use middleware for side effects
    - Implement selectors for derived state
